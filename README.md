## Introduction
In this task  hands-on learning in Security Operations Center practices. It includes threat hunting,post incident analysis,alert triage with automation,evidence analysis, adversary emulation, and security metrics. The goal is to simulate attacks, detect them, respond effectively, and improve SOC processes using open-source tools and structured methodologies.

## Key Learnings
- understood Proactive threat hunting using hypotheses and frameworks
- using Wazuh Detected attacks and suspicious activity and monitored logs, generated alerts. 
- TheHive is used Document and manage security incidents, triage alerts, track response actions.  
- Conducting post-incident analysis using 5 Whys and Fishbone diagrams to identify root causes.
- Simulating attacker behavior with MITRE Caldera to test detection and response.
- Measuring SOC performance using metrics like MTTD, MTTR, and dwell time.
- Communicating findings and recommendations effectively to stakeholders.

## Tools Used
- Wazuh – Threat detection and alerting.
- CrowdSec – Automated IP blocking and containment.
- Caldera – Adversary emulation and TTP simulation.
- TheHive – Incident documentation and triage.
- Elastic Security– SOC metrics visualization and dashboards.
- Google Docs & Sheets, Draw.io – Reporting, RCA, and diagram creation.
- Metasploit – Exploit simulation.

## Conclusion
This project provides practical experience in detecting, responding, and analyzing cybersecurity incidents. By combining tools, structured methodologies, and adversary simulations, it demonstrates how SOC teams can strengthen defenses, improve response times, and continuously enhance cybersecurity posture.
